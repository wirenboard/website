# Редактирование произвольных страниц

Страницы — похожи на статьи, но они не добавляются на страницу-аггрегатор «Статьи», а значит на них откуда-то должна быть ссылка: с другой страницы или из меню.

Инструменты необходимые для редактирования и предпросмотра описаны в [README.md](/README.md).

## Как добавить страницу

Перейдите в папку нужного языка:
* Английский [content/en/pages](/content/en/pages)
* Русский [content/ru/pages](/content/ru/pages)

Создайте пустой md-файл и рядом с таким же названием папку под картинки, например:
* `content/ru/pages/my_page.md` — файл с будущим текстом.
* `content/ru/pages/my_page` — папка под картинки. Сюда сложите картинки весом не больше 3 Мбайт каждая. Ограничений на разрешение картинок нет — движок сам обработает их в нужный размер. Формат картинок может быть `jpg`, `webp`, `png`, выбирайте удобный для вашей задачи.

Путь к картинкам указывать относительный, то есть без косой черты в начале, пример: `my_page/my_page.jpg`.
Для названий .md-файлов и картинок использовать латиницу.

Пробелы заменять на подчёркивание `_`. Не использовать тире, скобки, решётки и другие специальные.

Ниже шаблон md-файла:
```
---
title: Заголовок страницы
---

## Подзаголовок
Тут текст

### ПодПодзаголовок
Тут текст
```

Обратите внимание, что при использовании заголовков следует начинать с заголовка второго уровня (`##`) и далее, так как заголовок первого уровня будет автоматически сформирован из содержимого поля `title`.

В тексте можно использовать [компоненты](./components.md).

Изменения станут доступны для всех пользователей после отправки пуллреквеста и вливания утверждённых изменений.

## Предпросмотр при запуске сайта локально

Обычно для предпросмотра достаточно VSCode с плагином, но если вы запускаете сайт локально на компьютере, вот ссылка для предпросмотра: http://localhost:3000/ru/contents/PAGE_NAME, где нужно заменить `PAGE_NAME` на имя файла, например `wbce2025`.

## Публикация страницы на основном сайте (бекенд → Контент → Страницы)

Сам по себе md-файл доступен только по адресу нового фронта вида `/<язык>/contents/<имя-файла>` (например, `/ru/contents/cloud`). Чтобы страница появилась на основном сайте `wirenboard.com` под «нормальным» адресом (например, `wirenboard.com/pages/cloud`), её нужно дополнительно зарегистрировать в админке старого сайта.

Старый сайт умеет **подтягивать** HTML с нового фронта и выводить его в области контента своей страницы (в своём шаблоне — с шапкой, меню и подвалом). Решает за это одно поле — «Алиас внешнего контента»: если оно заполнено, контент берётся с нового фронта; если пустое — из поля «Основной контент» в самой админке.

### Что сделать

1. Сначала убедитесь, что md-файл уже влит и задеплоен на новом фронте: контент тянется в момент открытия страницы, и если файла ещё нет — область контента на основном сайте окажется пустой.
2. В админке откройте раздел **Контент → Страницы** и нажмите добавление новой страницы.
3. Заполните поля (важны прежде всего два алиаса):

| Поле | Что писать |
|------|------------|
| **Заголовок** | Заголовок страницы. |
| **Алиас** (`alias`) | Slug, под которым страница будет жить на основном сайте: `wirenboard.com/pages/<alias>`. Только латиница, цифры, `-` и `_` (шаблон `^[a-zA-Z0-9_-]+$`), до 100 символов. Можно задать вручную; если оставить пустым — сформируется автоматически из заголовка. |
| **Алиас внешнего контента** (`external_alias`) | Путь к контенту на новом фронте в формате `contents/<имя md-файла без .md>`. Для `content/ru/pages/cloud.md` → `contents/cloud`. До 64 символов. **Не указывайте язык и не ставьте слеши по краям** — язык подставляется автоматически, завершающий `/` добавляется тоже. |
| **Язык** (`language`) | Язык этой записи (`ru`/`en`). На каждый язык — своя запись «Страницы». |
| **Статус** | Поставьте «Опубликовано», иначе страница на фронте открываться не будет (фронт показывает только опубликованные записи). |
| **Шаблон страницы**, SEO-поля, **Основной контент** | Заполняются по необходимости. Поле «Основной контент» используется только как запасной вариант, когда «Алиас внешнего контента» пустой; при заполненном внешнем алиасе оно игнорируется. |

### Как формируется запрос к новому фронту

При открытии `wirenboard.com/pages/<alias>` старый сайт находит опубликованную запись по паре «алиас + язык» и, если «Алиас внешнего контента» заполнен, делает запрос:

```
<EXTERNAL_CONTENT_SERVICE_URL>/<язык>/<external_alias>/
```

Например, для записи с языком `ru` и `external_alias = contents/cloud`:

```
<адрес нового фронта>/ru/contents/cloud/   →   md-файл content/ru/pages/cloud.md
```

Язык в URL берётся из текущего языка страницы, поэтому для записей `ru` и `en` поле «Алиас внешнего контента» может быть **одинаковым** (`contents/cloud`) — старый сайт сам подставит нужный язык и подтянет соответствующий локализованный файл (`content/ru/...` или `content/en/...`).
